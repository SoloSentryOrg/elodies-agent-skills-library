# Security Policy

## Supported content

Security fixes apply to the current `main` branch. Historical skill versions are retained only through Git history and are not independently supported.

## Reporting

Use GitHub private vulnerability reporting for this repository. If that feature is unavailable, contact the repository owner through an approved private SoloSentry channel.

Do not disclose exploit details, malicious skill payloads, credentials, private prompts, or sensitive repository content in public issues, discussions, or pull requests.

## Scope

Reports may cover prompt injection, hidden instructions, unsafe tool invocation, approval bypass, arbitrary code execution, supply-chain compromise, secret exposure, data exfiltration, insecure updates, malicious external references, or governance-control bypass.

Preserve evidence safely and provide the affected skill/version, reproduction preconditions, impact, and suggested mitigation where possible.
